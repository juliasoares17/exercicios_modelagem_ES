# Exercícios de Fixação: Modelagem de Código e Engenharia de Software

Este repositório contém as resoluções dos exercícios práticos propostos pelo **Prof. Juan Hassan** na disciplina de Engenharia de Software. <br>
O objetivo principal destas atividades é exercitar a transição conceitual do design abstrato para a escrita segura de código, aplicando conceitos fundamentais do SDLC (Ciclo de Vida de Desenvolvimento de Software), TDD (Desenvolvimento Guiado por Testes) e Paradigmas de Software.

Todas as soluções são estruturadas de forma **agnóstica**, focando na robustez do design lógico e na arquitetura do domínio antes da implementação da lógica interna.

## 🚀 Tecnologias e Paradigmas
* **Paradigma:** Orientação a Objetos (POO)
* **Práticas:** Encapsulamento, TDD (Red-Green-Refactor), Design Incremental e Coesão.

## 📋 Estrutura dos Exercícios

| Exercício | Tema Principal | Conceitos Chave |
| :--- | :--- | :--- |
| **Ex 1: Design vs Código** | Mapeamento de Domínio | Fases Universais do SDLC, Multiplicidade, Controle de Estado |
| **Ex 2: Entrega Incremental** | Modelagem Modular | Fatiamento de Valor, Mitigação de Cascata, Simplicidade XP |
| **Ex 3: Validação com TDD** | Testes Dirigidos por Lógica | Blindagem Aritmética, Proteção Ativa, Red-Green-Refactor |
| **Ex 4: Simplicidade e XP** | Migração de Paradigma | Transição Procedural -> OO, Métodos Controlados, Refatoração |
